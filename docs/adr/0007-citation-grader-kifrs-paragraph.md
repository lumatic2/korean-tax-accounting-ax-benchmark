# 0007 — citation 코드채점을 기준서-문단(K-IFRS)까지 확장한다

## Status
Accepted (2026-06-09)

## Context
`grade_citation`(code_grader.py)은 결정론 근거채점의 핵심 경로다 — gold sources의 조문과
모델 답변이 인용한 조문을 매칭해 `legal_basis` 점수를 주고, gold에 없는 인용을
`unverified_citation`으로 플래그한다(환각 탐지). 이 경로가 M3 파일럿에서 모델 변별의
최대 축이었다(haiku 법인세 붕괴 = 가짜 조문 6건).

그러나 추출 정규식이 세무 조문만 안다:
- `_LOC_RE = 제\d+조(?:의\d+)?`, `_FAKE_HINT = 제\d+조`

회계(K-IFRS) 문항의 근거는 **조문이 아니라 기준서-문단**이다. 실데이터(sample-questions-v0.1):
- source: `title="K-IFRS 제1038호 무형자산"`, `locator="문단 57"`
- 모델 인용 규약(accounting 스킬): `[기준서-문단]` 예 `[1038-57]`

`문단 57`은 `제\d+조`에 안 걸려 **gold가 항상 빈 집합 → 회계 citation 문항은 legal_basis가
구조적으로 0점**(채점 불능). 이것이 blueprint §6 회계 ②근거 셀(잔여 3)을 막고 있었다.

추가로, source 측은 기준서 번호가 `locator`가 아니라 `title`("제1038호")에 있고, answer
측은 `[1038-57]` 한 토큰에 둘 다 있다 — **표기 형식이 비대칭**이라 단순 정규식 추가로는
매칭이 성립하지 않는다. 정규 매치키 설계가 필요하다.

선택지:
- **(A) judge에 위임** — 회계 근거는 LLM-judge가 채점. 결정론 앵커 포기(self-eval 천장 §2.2 재유입).
- **(B) 회계 전용 별도 채점기** — 함수 분기. 중복·유지보수 부담, 통합 플래그 체계 분산.
- **(C) grade_citation에 기준서-문단 매치키 통합** — 세무(`제N조`)·회계(`기준서-문단`)를
  하나의 정규 키 공간으로 흡수. 순수함수·결정론 유지.

## Decision
**(C).** 정규 매치키 = **`{기준서4자리}-{문단}`** (예 `1038-57`, `1115-B35`, `1109-4.1.2A`).

- **gold(source 측)**: `title`의 `제(\d+)호` + `locator`의 `문단\s*([0-9A-Za-z.]+)` → `1038-57`.
  세무 source는 기존대로 `_base_article`(`제25조`).
- **cited(answer 측)**: 1순위 대괄호 `[1038-57]`, 2순위 자연어 `제1038호 … 문단 57`(근접 윈도우).
  세무는 기존 `_LOC_RE`.
- **fake 탐지**: gold에 없는 인용 키 → `unverified_citation`(세무 `제999조의5`와 동일 구조).
  **순수함수 불변** — kifrs DB 조회로 기준서 실재 검증은 하지 않는다(네트워크·시간 금지,
  채점 결정론 유지). 구조적 미매칭만으로 충분히 환각을 잡는다(세무와 parity).
- 문항은 세무 또는 회계 둘 중 하나이므로 두 키 공간은 교차오염되지 않는다 — 함수 하나로 통합.

## Consequences
- ✅ 회계 citation 문항이 결정론 채점 + fake_source 탐지 대상이 됨 → blueprint §6 회계 ②근거
  셀(잔여 3) 차단 해제.
- ✅ 세무·회계 근거채점이 단일 경로·단일 플래그 체계(`unverified_citation`)로 통일.
- ✅ 순수함수·결정론 불변 — 기존 62 테스트 영향 없음(세무 키 경로 그대로).
- ⚠ 회계 fake 탐지는 *구조적*(gold에 없음)이지 *의미적*(존재하지 않는 기준서)이 아니다.
  존재하나 무관한 문단 인용은 `unverified`로만 잡고, 기준서 실재 검증은 judge/사람 몫.
- ⚠ 모델이 `[1038-57]` 규약을 안 쓰고 "무형자산 기준서 문단 57"처럼 기준서 번호를 생략하면
  키 합성 불가 → 미매칭. 자연어 fallback은 `제\d+호`가 있을 때만 동작(보수적, 의도적).
