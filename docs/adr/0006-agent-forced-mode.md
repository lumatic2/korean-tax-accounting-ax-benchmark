# 0006 — agent 도구 사용을 권위게이트 + 근거매칭으로 강제하는 `agent_forced` 모드를 둔다

## Status
Accepted (2026-06-09)

## Context
[ADR 0005] agent 러너 라이브 검증([findings/agent-tool-forcing.md](../findings/agent-tool-forcing.md))에서, 모델은 *기억으로 아는* 사실(유명 한도 등)에는 도구를 쓰지 않았다(3모델 전부 corp-tax-0012에서 도구 0회). 모델의 자발적 도구 사용에 의존하면 ⑤도구 측정이 불안정하다.

⑤도구 사용성은 두 하위 구성으로 갈린다: **판단**(도구가 필요함을 인지) vs **실행역량**(강제 시 올바른 도구를 정확히 호출·통합). 자유 모드(`agent`)는 판단을 재지만, 유명룰 문항에서 천장이 생긴다. 실행역량을 *재현 가능*하게 재려면 강제가 필요하다.

선택지:
- (A) 단순 카운트 게이트(도구 N회) — 간단하나 `계산` 1회로 우회 가능(게이밍).
- (B) 권위도구 게이트(`법령조문`/`기준서문단` ≥1, `계산` 불충분) — 우회 완화.
- (C) **권위게이트 + 근거매칭 코드점수** — (B) + 최종 답이 인용한 조문이 *실제 이번 run에서 조회된 것인지* 코드로 대조.

## Decision
**(C).** `benchmark_mode:["agent_forced"]` 신설:
- **게이트** — 러너 루프(`_run_agent`)가 `require_authority`일 때, 모델이 권위 도구(`법령조문`/`기준서문단`)를 1회도 안 쓰고 [최종]을 내면 거부하고 재촉(nudge ≤2회). 끝까지 미사용이면 flag `forced_tool_unmet`(-15).
- **근거매칭** — `grading/grounding.py`가 최종 답의 인용 조문/문단 토큰과 `agent_steps`에서 *성공 조회된* 토큰을 대조. forced 모드에서 인용은 있으나 grounded_ratio<0.5(=기억 인용)면 flag `ungrounded_citation`(-10). 결정론·재현 가능, `계산` 우회·엉뚱한 조회로는 점수를 못 얻는다.

`agent`(자유) 모드는 **존치**(판단 측정 + 개방형 문항). 둘은 별개 점수축.

CLI 훅이 아닌 **러너 게이트**로 강제한다(ADR 0005의 재현성 원칙 — CLI 훅은 버전핀 밖·stub 불가).

## Consequences
- ✅ ⑤도구 실행역량을 유명룰 여부와 무관하게 재현 가능하게 측정.
- ✅ 근거매칭은 기억-인용 vs 도구-근거를 코드로 가름 → 환각·게이밍 저항(judge 의존↓).
- ✅ 자유/강제 두 모드 = 두 하위 구성 분리 측정(M4 구성타당도).
- ⚠ **게이밍 잔존** — 무관한 권위 도구 1회로 게이트는 통과 가능. 근거매칭이 *점수*는 막지만(인용≠조회면 감점) 완전 차단은 아님. 정답 경로 설계(A')로 보완.
- ⚠ 근거매칭은 토큰 정규식 휴리스틱(제N조의M·[기준서-문단]·NNNN 문단 NN). 인용 형식이 어긋나면 누락 가능 — 보수적으로 forced 모드에서만 감점.
