# PRD

> 상위 비전은 [benchmark-design-principles.md](benchmark-design-principles.md), 공개 범위는 [m4-public-sample-scope.md](m4-public-sample-scope.md)를 따른다. 이 문서는 실행기(M2~M3) MVP 범위만 규정.

## 목표
한국 회계·세무 AI가 "그럴듯하게 말하는가"가 아니라 **"실무 검증을 통과했는가"**를, 같은 조건에서 재현 가능하게 측정하는 평가 실행기.

## 사용자
- **개발자 본인(1차)** — 포트폴리오 증명 + 창업 검증. 문항 제작·검수·실행·리포트 전 과정 운용.
- **회계법인 AI 부서(M5)** — 벤더·자사 에이전트를 줄 세우는 잣대로 기업 평가 리포트 구매.

## 핵심 기능 (MVP = M2~M3)
1. **3모드 실행기** — 한 문항을 closed_book / rag / agent 모드로 동일하게 돌려 모델 응답을 버전핀과 함께 수집.
2. **다차원 채점** — 코드 채점(MC·계산·근거, 결정론) + LLM-judge(사례·근거, 루브릭 7차원) + statement-level 부분점·pass^k.
3. **진단 리포트** — 3~5개 모델을 분야×차원으로 분해해 "어디가 약한지"(환각·계산오류·근거오류)를 보이는 변별 리포트.

## MVP 제외 사항
- **공개 리더보드·canary**(M4) — 신뢰 자산이 쌓인 뒤. 지금은 private repo.
- **외부 전문가 검수**(M4+) — 1차는 본인 단독 검수.
- **PDF/HWPX 기출 파서** — 문항은 시드→재작성 수작업(저작권 안전선). 자동 파싱은 범위 밖.
- **fine-tune·자체 모델** — 평가 대상이지 학습 대상이 아님.

## 성공 지표
- **M2**: 한 모델을 3모드로 돌려 7차원 점수 산출. **같은 입력 → 같은 코드 점수**(자동채점 결정론, pytest 강제). 버전핀(모델·데이터 hash·스캐폴드) 기록.
- **M3**: 주요 모델 3~5개 점수가 **유의미하게 벌어짐**(saturation/floor 아님). 분야×차원 진단 리포트 1건 산출.
