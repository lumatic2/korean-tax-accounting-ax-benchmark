# K-TaxBench 자료 출처·저작권 맵 v0.1

## 목적

K-TaxBench는 신뢰 인프라를 목표로 하므로, 문항의 출처와 저작권 상태를 처음부터 관리해야 한다.

핵심 원칙:

1. **기출문제·교재 원문을 그대로 복제하지 않는다.**
2. 법령·판례·공공자료는 기준시점과 URL을 남긴다.
3. 실무사례는 개인정보와 회사 식별정보를 제거한다.
4. 공개셋과 비공개셋을 분리한다.
5. 모든 문항에 `source_copyright_risk`와 `reuse_policy`를 기록한다.

## 출처 분류

## 1. 법령

### 예시 자료

- 국가법령정보센터
- 법제처 Open API
- 국세기본법
- 소득세법
- 법인세법
- 부가가치세법
- 조세특례제한법
- 상법
- 외부감사법

### 활용 방식

- 조문 근거 연결
- 기준시점별 법령 확인
- RAG 평가용 근거 문서

### 저작권/리스크

- 위험도: 낮음
- 공개 가능성: 높음
- 주의: 개정 시점 관리 필수

### 메타데이터 필수값

- 법령명
- 조문 번호
- 기준일
- URL 또는 API 식별자
- 접근일

## 2. 시행령·시행규칙

### 예시 자료

- 부가가치세법 시행령
- 법인세법 시행령
- 소득세법 시행령
- 국세기본법 시행령

### 활용 방식

- 세부 요건 판단
- 한도/비율/절차 기준 확인

### 저작권/리스크

- 위험도: 낮음
- 공개 가능성: 높음
- 주의: 법률보다 자주 바뀌는 세부 규정 관리 필요

---

## 3. 판례

### 예시 자료

- 대법원 판례
- 하급심 판례
- 조세 관련 행정소송 판례

### 활용 방식

- 사례 판단형 문항
- 사실관계 비교 문제
- 결론/근거 추론 평가

### 저작권/리스크

- 위험도: 낮음~중간
- 공개 가능성: 중간~높음
- 주의: 판례 전문 제공 범위와 출처 정책 확인 필요

### 문항화 원칙

- 판례 사실관계를 그대로 길게 복제하기보다 요약·변형한다.
- 사건번호, 선고일, 핵심 쟁점을 기록한다.
- 유사하지만 결론이 달라지는 변형 사례를 만들 수 있다.

### 저작권/리스크

- 위험도: 낮음~중간
- 공개 가능성: 중간
- 주의: 개인정보·사업자 식별정보 제거 필요

- 국세청 질의회신
- 기획재정부 해석
- 사전답변

### 저작권/리스크

- 위험도: 낮음~중간
- 공개 가능성: 중간
- 주의: 최신성, 폐기/변경 여부 확인 필요

## 6. 회계기준

### 예시 자료

- K-IFRS
- 일반기업회계기준
- 중소기업회계기준
- 한국회계기준원 공개 자료

### 활용 방식

- 회계처리 판단 문항
- 분개/표시/공시 판단
- 수익인식, 리스, 금융상품 등 고난도 문항

### 저작권/리스크

- 위험도: 중간
- 공개 가능성: 제한적
- 주의: 기준서 원문 복제 범위 확인 필요

### 문항화 원칙

- 기준서 전문 복제보다 요건을 요약하고 출처를 연결한다.
- 공개셋에는 원문 장문 인용을 피한다.

---

## 7. 시험 기출문제

### 예시 자료

- 공인회계사 시험
- 세무사 시험
- 재경관리사/회계관리 등 자격시험

### 활용 방식

- 기본 지식·계산형 문항 구조 참고
- 난이도 기준 참고
- 공개 시험형 벤치마크 구성

### 저작권/리스크

- 위험도: 중간~높음
- 공개 가능성: 낮음~중간
- 주의: 원문 복제 금지 원칙

### 문항화 원칙

- 원문을 그대로 저장하지 않는다.
- 개념·구조·난이도를 참고해 신규 문항을 만든다.
- 출처에는 “inspired_by” 또는 “exam_reference”로 기록한다.
- 공개셋에는 특히 변형도가 높은 자체 제작 문항만 포함한다.

## 8. 실무 사례

### 예시 자료

- 스타트업 세무 이슈
- 프리랜서/개인사업자 사례
- 법인 비용처리 사례
- 부가세 신고 오류 사례
- 세무조정 체크리스트

### 활용 방식

- 사례 판단형 문항
- 리스크 분석형 문항
- 실무 커뮤니케이션 평가

### 저작권/리스크

- 위험도: 중간
- 공개 가능성: 자체 제작이면 높음
- 주의: 개인정보, 회사명, 금액, 거래처 정보 비식별화 필수

### 문항화 원칙

- 실제 사례는 합성/익명화한다.
- 특정 회사가 식별되지 않게 한다.
- 공개 전 민감정보 점검을 거친다.

### 활용 방식

- 유료 교재·강의자료 문항은 사용하지 않는다.
- 아이디어 차원의 참고와 원문 복제를 구분한다.

---

## 공개 가능성 기준

### 공개 가능성이 높은 문항

- 자체 제작 문항
- 법령/판례 근거를 바탕으로 새로 구성한 사례
- 개인정보 없는 합성 사례
- 원문 복제 없는 요약형 문항

### 공개 보류 문항

- 기출 구조와 유사도가 높은 문항
- 회계기준 원문 인용이 많은 문항
- 전문가 검수 전 고난도 실무 문항

### 공개 금지 문항

- 유료 교재/해설을 기반으로 한 문항
- 실제 고객 사례가 식별 가능한 문항
- 저작권 상태가 불명확한 원문 복제 문항
- 비공개 평가셋/홀드아웃 문항

## 리스크 등급

### Low

- 법령 중심
- 자체 제작 사례
- 공개 판례 요약
- 원문 복제 없음

### Medium

- 시험 기출의 구조를 참고
- 회계기준 일부 요약
- 공개 질의회신·심판례 요약
- 실무 사례를 익명화

### High

- 유료 교재/해설 의존
- 기출문제 원문과 유사
- 회사/개인 식별 가능
- 출처 불명확

- 학술 논문

### 3순위

- 실무자 인터뷰
- 자체 합성 사례
- 기업 파일럿에서 익명화된 오류 사례

## 운영 원칙

1. 첫 MVP는 **자체 제작 + 공개 법령/판례 기반**으로 만든다.
2. 기출문제는 난이도와 범위 참고용으로만 사용한다.
3. 비공개 평가셋은 공개셋보다 엄격히 관리한다.
4. 모든 문항은 기준시점과 출처를 가진다.
5. 공개 전 저작권·개인정보·전문가 검수 상태를 확인한다.
