# 데이터 전략 (K-TaxBench)

> 무슨 자료를 어떻게 모으고·정리하고·보호하고·버전관리하는가. [benchmark-design-principles.md](benchmark-design-principles.md)의 load-bearing 3축 중 **오염저항·적시성**과 **표현충실성(전문가 정답)**이 여기서 실현된다.
> 상태: **v0.1 (2026-06-01)** — 한국 저작권 1차 리서치 반영. 법률 자문이 아니라 내부 운영 기준이며, 상업화 전 전문가(변호사) 검토 권장.

---

## 0. 운영 원칙 (한 줄)

> **기출문제는 쟁점·유형 추출용 *시드(seed)*로만 쓰고, 표현(사실관계·수치·문장)은 전부 새로 작성한다.**

이 한 줄이 세 문제를 동시에 푼다 — 저작권 침해 회피 + 학습누출(오염) 회피 + 현행 법령 기준(적시성). 단순 "기출 재사용"이 아니라 **시드 기반 재작성**이다.

## 1. 자료 3층 구조

| 층 | 내용 | 저작권 지위 | 용도 |
|---|---|---|---|
| **1층. 정답 근거 (공개 1차 자료)** | 법령·시행령·시행규칙, 국세청 고시·예규, 판례, 조세심판원 결정 | **§1 저작권 보호 대상 아님(저작권법 제7조)** — 자유 이용 | 정답·해설의 *근거* 박제 |
| **2층. 문항 소재 (참고)** | CPA·세무사 기출, 실무 사례 | **시드로만** — 표현 복제 금지(§2) | 쟁점·유형·난이도 추출 |
| **3층. 생산물 (우리 저작물)** | 문항·정답·해설·루브릭 | **우리 소유** | 공개/판매 권리 확보 |

## 2. 저작권 (제품의 생사)

### 2.1 1층은 안전 — 저작권법 제7조

저작권법 **제7조(보호받지 못하는 저작물)**: ① 헌법·법률·조약·명령·조례·규칙 ② 국가·지자체의 고시·공고·훈령 그 밖에 유사한 것 ③ 법원의 판결·결정·명령·심판, 행정심판 의결·결정 등 ④ 위의 편집물·번역물(국가·지자체 작성) ⑤ 사실 전달에 불과한 시사보도.

→ **법령·판례·조세심판원 결정·국세청 고시는 자유 이용.** 정답 근거를 1차 자료에 직접 연결하면 저작권 문제 없음. (단 **판례평석·해설서·문제집**은 창작성 있는 저작물 → 인용·복제 주의.)

### 2.2 2층(기출)은 위험 — 시드로만

- 시행기관(한국교육과정평가원 등)은 기출 저작권을 명시 보유, 무단복제·배포·출판 금지. **영리 목적 대량복제는 침해 소지**(대법원 시각). CPA(금융감독원)·세무사(한국산업인력공단) 시행 문제도 동일하게 볼 여지.
- 그러나 저작권은 **표현**에 미치고 **아이디어(쟁점·법리·계산 구조)에는 미치지 않는다**(아이디어/표현 이분법). 대법원 창작성 기준상 "누가 해도 같거나 비슷한 표현"은 창작성이 약함. 시험문제 저작권은 "질문의 표현·답안 표현·요소의 선택·배열·조합의 창작성"에 한정.
- **운영 규칙**: 기출의 *쟁점·유형·난이도*만 추출(아이디어), 지문·문장·수치·선택지는 **전면 재작성**(표현 신규 생산). 원문 표현을 그대로 옮기지 않는다. 기존 저작물(시·소설 등) 인용 출제분은 재인용 금지(원저작권 침해 위험).

### 2.3 공개 데이터 라이선스 — 공공누리(KOGL)

4유형: **제1유형**(출처표시 — 상업적 이용·2차적저작물 작성 자유), 제2유형(+상업금지), 제3유형(+변경금지), 제4유형(+상업·변경 금지). 출처표시는 전 유형 필수.

→ 우리는 유료 리포트 등 **상업화** 계획이므로 **제1유형 자료 우선**. 제2·4유형은 별도 이용허락 필요. (단 법령·판결 *원문*은 제7조로 KOGL과 무관하게 자유.)

### 2.4 [검증 필요]

- 국세청 **예규·질의회신**이 제7조 ②"고시·공고·훈령 유사물"에 해당하는지 (해당하면 자유).
- 국가법령정보센터·국세법령정보시스템 **Open API 이용약관**의 재배포·상업이용 제한 여부 (API ToS는 저작권과 별개로 계약상 제한 가능).
- CPA·세무사 **시행기관별 기출 이용정책** 실제 문구.
- 상업화 직전 **변호사 검토** (특히 비공개 판매셋).

## 3. 문항 제작 파이프라인

```
기출/실무 시드 (쟁점·유형 추출)
  → 1층 근거 연결 (법령·판례 조문 매핑, 기준일 확정)
  → 표현 전면 재작성 (지문·수치·선택지 신규)
  → 정답·해설·루브릭 작성
  → 동료/전문가 검수 (세무사·회계사)   ← 표현충실성 게이트
  → 기준일 도장 + 메타데이터 태깅
  → 공개/비공개 라우팅 + 해시 등록
```

- 검수 없는 문항은 비공개 검증셋 편입 금지.
- 스키마는 [benchmark-schema.md](benchmark-schema.md)·[rubric-v0.1.md](rubric-v0.1.md) 따름. (필요 시 기준일·근거링크·시드출처·해시 필드 추가 검토)

## 4. 오염 방지 (검증가능성 직결)

- **post-cutoff 신규 문항**: 모델 학습 시점 이후 개정세법 기반 → 학습누출 불가 + 신선도. (시드 재작성 원칙과 자연 결합)
- **canary 문자열**: 공개셋에 고유 문자열 삽입 → 향후 학습데이터 오염 추적.
- **해시 추적**: 문항별 해시 등록, 공개 시점 기록.
- **비공개셋 로테이션** + **제출 횟수 제한** (변형 fishing·과적합 차단 — 「Leaderboard Illusion」 교훈).

---

## 5. 공개/비공개 분리 운영

design-principles §4 표를 따름. 추가 운영 규칙:
- 공개 샘플(10~20%)은 오염 *전제* → "연습문제", 주기 교체.
- 비공개 검증셋(80~90%)은 점수·리포트·배지 근거 → 접근 통제·로테이션.
- 공개↔비공개 간 문항 누수 금지 (해시 대조).

- 문항·정답마다 **기준일**(`YYYY-MM-DD 시행 법령 기준`).

## 7. MVP 데이터 계획 (부가세 + 법인세)

- **범위**: 부가가치세, 법인세 우선 (실무 빈도·AX 수요·계산/세무조정/근거 풍부 — README MVP 정합).
- **문항 수**: 파일럿 50~100 (design-principles §7: 50~100은 파일럿 OK, 분야별 신뢰구간 주장하려면 주요 분야별 ~100+ 필요).
- **유형 배분**: 객관식/계산형/사례형/리스크형 (positioning·README 비율 참고, 채점 분산 측정 후 조정).
- **착수 순서**: ① 부가세 1층 근거 카탈로그(config/sources.md) → ② 시드 수집·쟁점 추출 → ③ 표현 재작성·근거 연결 → ④ 검수 → ⑤ 공개/비공개 라우팅.
