## 0. 왜 이 문서가 필요한가

M3 1차 파일럿(2026-06-02)에서 모델 변별(spread 36)은 **환각·근거·판단**에서 나왔지 **산수에서 나오지 않았다.** 법인세에서 haiku가 붕괴(13 vs opus 66.5)한 원인은 *가짜 조문 6건*이었다. 즉 이 벤치마크가 재야 하는 능력은 "계산을 맞히는가"가 아니라 **"근거를 지어내지 않고, 올바른 룰을 골라, 불확실성을 정직하게 다루는가"** 다.

설계 원리 §1.1은 이미 *"객관식 정답률이 높다고 세무조정을 맡길 수 있는 게 아니다… 단순 지식 암기형은 기초 변별용으로만, 실무 타당도는 사례·agent형이 담당"* 이라 못박았고, §5 안티패턴은 *Saturation*(다 만점→변별 0)과 *구성 무효*(MCQ가 실무 못 대변)를 경고한다. **이 문서는 그 원칙을 확충 가능한 비중표로 번역한 것이다.**

## 1. 유형별 구성 목표

각 `task_type`은 *실무 실패모드*의 프록시다. 비중은 "실무 신뢰도를 가르는 정도 × 채점 신뢰성"으로 매긴다.

| task_type | 측정하는 실무 실패모드 | 채점 경로 | 목표 비중 |
|---|---|---|---|
| **citation**(근거제시) | **환각** — 가짜 조문/판례 (세무에선 곧 배상책임). 최대 변별축 | code(조문매칭) + judge | **25~30%** |
| **case_reasoning**(사례판단) | facts → 어떤 룰을 적용하는가 (IRAC) | judge + 스팟체크 | ~25% |
| **risk_analysis**(리스크) | **불확실성 정직성** — "자료 부족, 추가 요청" vs 지어냄 | judge | 15~20% |
| **agent_workflow**(에이전트) | 도구 오케스트레이션·누락자료 요청·다단계 | pass^k | 10~15% |
| **calculation**(룰프록시 한정) | 한도/율/조건분기 숙련 (산수 ❌) | code(결정론 앵커) | ~10% |
| **multiple_choice / short_answer** | 기초 변별·캘리브레이션 | code | 5~10% |

> 비중은 분야(domain) 안에서가 아니라 **전체 풀 기준**의 지향점이다. 분야별로 적합한 유형이 다르다(예: agent는 mixed·복합사례에 몰림).

## 2. 계산문항(calculation) 채택 기준 — 핵심 규칙

> **단일 판정 질문: "이 문항을, 룰을 아는 전문가도 *룰 선택을 잘못해서* 틀릴 수 있는가?"**
> - **YES** → 채택. 숫자는 룰 숙련도(한도·율·요건·조건분기)의 *결정론적 프록시*. 코드 채점이 싸고 재현가능(judge·self-eval 리스크 없음, §2.2)하면서 변별한다.
> - **NO**(산수만 물음) → **배제.** 도구 가진 AI 시스템(=실제 AX 제품)은 산수를 틀리지 않는다 → Saturation.

계산문항의 4겹 중 **2~4번에 난이도가 있어야** 한다:
1. 산수(덧셈·곱셈) — AI 안 틀림 ❌
2. 어떤 공식/한도/율을 쓸지 선택 ✅
3. 조건 분기(예: 운행기록부 작성여부 → 비율 → 한도) ✅
4. 사실관계에서 입력값 추출 ✅

**본보기**: `corp-tax-0003`(기업업무추진비 한도 — 중소기업 오인·구간 누적식 함정), `corp-tax-0004`(업무용승용차 — 운행기록부 미작성 시 비율 분기), `ktb-vat-0003`(간이과세 — 두 임계값 구분).

## 3. 유형을 가로지르는 두 속성 (모든 문항에 심는다)

- **함정(plausible-wrong)** — 그럴듯한 오답이 존재해야 약한 모델이 패턴매칭으로 틀린다. `common_wrong_answers`·객관식 오답지·`fatal_errors`로 명시. 본보기: `vat-0004`(영세율≠면세), `vat-0010`(미가공≠가공), `corp-0001`(접대비 vs 광고선전비), `vat-0003`(면제≠신고불요).
- **적시성(timeliness, §2.3)** — `time_basis` 도장 필수. 가능하면 2026 현행 vs 개정 전이 갈리는 문항(post-cutoff = 오염저항 + 변별 + 우리 해자).

## 4. positioning 5대 역량 ↔ task_type 매핑

[positioning.md](positioning.md)의 5대 평가축을 유형으로 환산(진단 리포트·M5 기업 리포트의 역량별 점수 축):

| 5대 역량 | 주 담당 task_type |
|---|---|
| ① 정확성(결론·계산) | calculation, multiple_choice |
| ② 근거성(조문·판례 인용) | **citation**, case_reasoning |
| ③ 실무성(쓸 수 있는 답) | case_reasoning, agent_workflow |
| ④ 리스크 인식(불확실성 구분) | **risk_analysis**, case_reasoning |
| ⑤ 도구 사용성(RAG/Agent 검증) | **agent_workflow** (rag/agent 모드) |

> 문항에 "측정 역량"을 명시하는 `measures` 필드를 스키마에 추가하는 안은 **deferred**(hash 산출 영향·scope). 당장은 `tags`와 본 매핑으로 갈음.

## 6. 토픽 커버리지 매트릭스 (주제 × 역량, 목표 ~100)

§1·§4가 *형식·역량 비중*을 정했다면, 이 매트릭스는 **어떤 주제를 덮어야 "한국 세무·회계 실무 역량"을 신뢰성 있게 주장할 수 있는가**(구성 타당도의 주제 축)를 고정한다. 확충은 이 지도를 *채우는* 작업이며, M4 방법론 리포트의 커버리지 근거가 된다.
