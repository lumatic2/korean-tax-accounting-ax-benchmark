# K-TaxBench Leaderboard

한국 회계·세무 AI의 **실무 신뢰도**를 측정하는 벤치마크 **K-TaxBench**의 공개 리더보드입니다.

> **라이브: [tax-benchmark.askewly.com](https://tax-benchmark.askewly.com)**

## 이 저장소는

리더보드 웹앱의 **정적 빌드 산출물**(Next.js static export)만 담아 GitHub Pages로 서빙합니다. 벤치마크 문항·평가 실행기·채점 루브릭 등 소스는 별도의 비공개 저장소에서 관리합니다. 따라서 이 저장소의 파일은 빌드할 때마다 통째로 교체됩니다.

## K-TaxBench란

"AI가 그럴듯하게 말하는가"가 아니라 **"실무 검증을 통과했는가"**를 재는 평가 인프라입니다. 한국 세무·회계 실무를 101문항(부가세·법인세·소득세·회계처리·기초세법·복합)으로 분해하고, 생성형 답변을 다섯 축으로 채점합니다.

- **정확성** — 결론과 계산이 맞는가
- **근거성** — 조문·기준서·예규·판례 인용이 맞는가 (조문·문단 단위 코드 채점)
- **실무성** — 세무사·회계사가 그대로 검토할 수 있는 형태인가
- **리스크 인식** — 불확실성과 추가 확인사항, 위험 조건을 구분하는가
- **도구 사용성** — RAG/Agent가 자료 검색과 검증을 제대로 수행하는가

평가는 **closed-book / RAG / agent** 세 모드로 실행됩니다. 평가받는 모델이 자기 답을 채점하지 못하도록 심판은 후보군 밖 모델로 두고, 후보·심판의 실행 환경을 격리합니다.

## 리더보드 정책

리더보드 점수 왜곡(재시도 후 최고점만 공개, 불리한 결과의 선택적 철회, 접근 비대칭, 공개셋 과적합)을 막기 위해 다음 규칙을 적용합니다.

- **버전 핀 동결** — 모델 id·데이터 해시·스캐폴드·모드를 점수에 함께 기록
- **holdout 순위 우선** — 비공개 검증셋 순위를 기준으로 표기, 공개셋 점수는 별도
- **철회 불가** — 등재 결과는 supersede만 가능
- **재현 검증 등재** — self-report가 아니라 동일 공개셋·동일 실행기로 재현된 결과만

자세한 정책은 리더보드 화면의 정책 배지와 푸터를 참조하세요.
